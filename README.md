    <div class="section-3"  id="11567147_SilkPerformerPlugin-HowtousetheSilkPerformerPlugin"  >
        <h3>How to use the SilkPerformer Plugin</h3>
    <p>
The SilkPerformer Plugin offers the ability to record a session when a load test (even a try script) is started. This can be done by setting the option <tt class=" ">&quot;Automatic Session Recording&quot;</tt> in the Plugins <tt class=" ">&quot;Attributes&quot;</tt> tab to true.    </p>
    <p>
The <tt class=" ">&quot;Wait Timeout&quot;</tt> Attribute determines the time the SilkPerformer is waiting for the dynaTrace Client to be started. If the dynaTrace Client cannot be reached within this interval the load test will start without dynaTrace session recording.    </p>
    <p>
The <tt class=" ">&quot;System Profile&quot;</tt> Attribute determines the dynaTrace System Profile on which session recording should be started. If omitted, session recording will be started on the first System Profile on the dynaTrace Server with an Agent connected.    </p>
    <p>
The <tt class=" ">&quot;SilkPerformer Diagnostics Client Port&quot;</tt> Attribute determines the dynaTrace client REST interface port.    </p>
    <div class="confbox admonition admonition-note">
    <p>
If the dynaTrace Client is connected to multiple dynaTrace servers the first connected server will be used.    </p>
    </div>
    <div class="confbox admonition admonition-note">
    <p>
If you change the configured server in dynaTrace Client SilkPerformer has to be restarted.    </p>
    </div>
    </div>
    </div>

    <div class="section-3"  id="11567147_SilkPerformerPlugin-HowtousetheSilkPerformerPlugin.1"  >
        <h3>How to use the SilkPerformer Plugin</h3>
    <p>
The dynaTrace Plugin offers the ability to record a session when a load test (even a try script) is started. This can be done by setting the option <tt class=" ">&quot;Automatic Session Recording&quot;</tt> in the Plugins <tt class=" ">&quot;Attribute&quot;</tt> tab to true.    </p>
    <p>
The <tt class=" ">&quot;Wait Timeout&quot;</tt> Attribute determines the time the SilkPerformer is waiting for the dynaTrace Client to be started. If the dynaTrace Client cannot be reached within this interval the load test will start without dynaTrace session recording.    </p>
    <p>
The <tt class=" ">&quot;Application Session&quot;</tt> Attribute determines the dynaTrace System Profile on which session recording should be started. If omitted, session recording will be started on the first System Profile on the dynaTrace Server with an Agent connected.    </p>
    <div class="confbox admonition admonition-note">
    <p>
If the dynaTrace Client is connected to multiple dynaTrace Servers there may be problems with matching the correct System Profile. If the System Profile is available on multiple dynaTrace Servers, it is possible that session recording is started on the wrong Server. If the <tt class=" ">&quot;profile name&quot;</tt> attribute is omitted, session recording may be started on any of the connected dynaTrace Server on the first System Profile with an Agent connected.    </p>
    </div>
